classi e oggetti
...

sono in grado di
-modellare gli oggetti nel mondo reale
rappresentare oggetti fisici
rappresentare concetti astratti
rappresentare entità software
rappresentare stati di un processo

definizione
...

una #classe è un pezzo del codice sorgente di un programma che descrive un particolare tipo di oggetti
Le classi vengono definite dal programmatore
La classe fornisce un prototipo astratto per gli oggetti di un particolare tipo
viene definita la struttura in termini di

  • #campi stato degli oggetti
  • #metodi comportamenti degli oggetti
    un #oggetto è un'istanza di una classe
    un #programma può creare e usare più oggetti della stessa classe

#classe
...

  • definita mediante parte del codice sorgente del programma
  • scritta dal programmatore
  • specifica la struttura dei campi dei suoi oggetti
  • specifica il comportamento dei suoi oggetti mediante il codice dei metodi

#oggetto
...

  • un'entità all'interno di un programma in esecuzione
  • creato quando un programma "gira"
  • contiene specifici valori dei campi (i valori possono cambiare durante l'esecuzione)
  • si comporta nel modo prescritto dalla classe quando il metodo corrispondente viene chiamato a tempo di esecuzione

ogni #classe è memorizzata in un file separato
il nome del file DEVE essere lo stesso della classe con estensione.java
i nomi di classe iniziano sempre con una maiuscola
i nomi in java sono case-sensitive

librerie
...

  • i programmi java si forniscono di diverse librerie nelle quali è scritto il codice per poter svolgere determinate funzioni
  • le #classi sono organizzate in #package
  • un package speciale è java.lang, esso contiene le classi fondamentali per la programmazione in java

un #campo costituisce la memoria privata di un oggetto
ogni #campo ha un tipo di dati
ogni #campo ha un nome fornito dal programmatore
la dichiarazione di un #campo avviene come segue:
#private [#static] [final] #tipo_di_dati nome;
#private tipicamente di accesso privato
#static se specificato indica che il campo è condiviso da tutti
#final indica che il campo è una costante

#metodi
...

un #metodo è tipicamente #pubblico , ovvero visibile a tutti
il nome di un metodo per convenzione inizia sempre con una lettera minuscola mentre le parole successive iniziano sempre con una lettera maiuscola #CamelCase

la definizione di un metodo avviene come segue
#public #tipo_di_dati nomeDelMetodo ( #tipo_di_dati nomeParametro ....){
istruzione1
istruzione2
...
}
un metodo nel caso in cui specificato il #tipo_di_dati deve restituire un valore al chiamante
nel caso in cui è specificato sia #void non restituisce nulla

#costruttori
...

  • #metodi per la creazione di oggetti di una classe
  • possiedono sempre lo stesso nome della classe
  • inizializzano i campi dell'oggetto
  • possono prendere zero, uno o più parametri
  • non hanno valori di uscita
  • una classe può avere anche più costruttori che differiscono nel numero e nei tipi dei parametri
  • non è obbligatorio dichiarare un costruttore
  • se ne viene specificato uno, java crea per ogni classe un costruttore di default "vuoto"
    • inizializza le variabili d'istanza ai valori di default

se per esempio abbiamo due costruttori della classe Contatore

static public void Contatore(){
	int contatore
	public void Contatore (){this.contatore = 0}
	public void Contatore (int contatore){this.contatore = contatore}
}

se noi nel main richiamassimo :

	Contatore contatore1 = new Contatore() // richiameremmo il primo costruttore che non accetta parametri 
	Contatore contatore1 = new Contatore(42) //richiameremmo il secondo contatore che accetta un parametro

variabili locali vs. campi
...

i #campi sono variabili dell'oggetto

  • sono visibili almeno all'interno di tutti gli oggetti della stessa classe
  • esistono per tutta la vita di un oggetto
    le #variabili_locali sono variabili definite all'interno di un metodo
  • come parametri del metodo o all'interno del corpo del metodo
  • esistono dal momento in cui sono definite fino al termine dell'esecuzione della chiamata al metodo in questione

qui possiamo vedere come si interpretano le classi all'interno di un linguaggio detto:
UML

valori primitivi vs. oggetti
...

è importante tenere a mente la differenza tra

  • #valori di tipo #primitivo (int, char, bool, float, ecc.)
  • #oggetti (istanze delle classi)

la loro rappresentazione in memoria è differente :

  • #valori_primitivi: memoria allocata automaticamente a tempo di compilazione
  • #oggetti: memoria allocata durante l'esecuzione del programma (operatore #new)

al momento della creazione dell'oggetto i campi di una classe sono inizializzati automaticamente

tipo del campoinizializzato implicitamente a
int, long0,0L
float, double0.0f , 0.0
char'\0'
booleanfalse
classe Xnull

riferimenti e oggetti
...

un #riferimento è un indirizzo di memoria, tuttavia non conosciamo il valore numerico dell'indirizzo
quindi gli oggetti non sono mai memorizzati direttamente nelle variabili, ma solo mediante il loro #riferimento

le variabili contengono :

  • valori di tipi di dati primitivi
    oppure
  • riferimenti a oggetti

    non esistono variabili che contengono oggetti

assegnazione

package
...

  • le classi vengono inserite (categorizzate) in collezioni dette package

  • ogni package racchiude classi con funzionalità correlate

  • quando si utilizza una classe è necessario specificarne il package

  • le classi che abbiamo utilizzato finora per esempio, appartengono al package speciale java.lang che non ha bisogno di essere specificato

  • per evitare di specificare il package di una classe ogni volta che viene usata è sufficiente importare la classe tramite il comando "import"

  • per importare un intero package possiamo scrivere per esempio import java.util.*

  • i package sono rappresentati fisicamente in cartelle

  • una classe può essere inserita in un determinato package semplicemente

    • specificando all'inizio del file (#package )
    • posizionando il file nella corretta sottocartella
